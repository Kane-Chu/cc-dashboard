# cc-dashboard 详细设计

## 1. 服务端设计（Node.js）

### 1.1 数据收集流程

```
sessions/*.json ──► getSessions() ──► collectData()
                              │
                              ▼
                    projects/*/<session-id>.jsonl
                              │
                              ▼
                         parseTranscript()
                              │
                              ▼
                    status / tokens / messages / pendingTools
```

### 1.2 状态判定逻辑

```
最后一条 assistant 消息 stop_reason == 'tool_use'
    └──► status = 'waiting'

最后一条消息是 user
    └──► status = 'running'（Claude 正在处理）

最后一条消息是 assistant 且无 stop_reason
    └──► status = 'running'（仍在流式输出）

其他情况
    └──► status = 'idle'
```

### 1.3 远程按键注入

通过 AppleScript 实现：
1. `ps -o tty= -p <pid>` 获取进程 TTY
2. 遍历 Terminal.app / iTerm2 窗口查找匹配 TTY
3. `System Events` 发送按键 + Return

限制：VS Code 集成终端、Warp 等无 TTY 或无法通过 AppleScript 访问的终端不支持。

### 1.4 SSE 消息流

```
客户端连接 /api/sessions/:id/stream
    │
    ├──► 发送最近 50 条历史消息（过滤命令类注入内容）
    │
    ├──► 每秒检查 transcript 文件变化
    │    └──► 有新行时推送 message 事件
    │    └──► 文件截断时推送 reset 事件
    │
    ├──► 每 15 秒发送 ping 心跳
    │
    └──► 进程结束时推送 close 事件
```

## 2. iOS App 设计

### 2.1 架构

```
App Entry (cc_dashboard_iosApp.swift)
    └──► SessionListView (主界面)
            ├──► SessionCardView (卡片)
            ├──► SettingsView (设置 Sheet)
            └──► ConfirmSheetView (确认操作 Sheet)

Models: Session, SessionStatus, SessionSource, Message, PendingTool
Services: DashboardAPI, SettingsStore
```

### 2.2 视图层级

```
NavigationStack
    ├── ZStack
    │     ├── List (sortedSessions)
    │     │     └── SessionCardView × N
    │     └── ProgressView (空状态加载)
    ├── Toolbar (settings-button)
    ├── .sheet (SettingsView)
    ├── .sheet (ConfirmSheetView)
    ├── .task (初始加载)
    ├── .task (自适应轮询)
    └── .alert (错误提示)
```

### 2.3 轮询策略

```swift
var pollingInterval: TimeInterval {
    hasWaitingSession ? 2.0 : 5.0
}
```

`.task(id: hasWaitingSession)` 在 `hasWaitingSession` 变化时自动取消旧任务、启动新任务。

### 2.4 网络层

```swift
DashboardAPI
    ├── fetchSessions(baseURL:) → [Session]
    ├── sendAction(baseURL:sessionId:action:) → ActionResponse
    └── testConnection(baseURL:)
```

URL 验证：scheme 白名单（http/https）+ host 非空检查。

### 2.5 设置存储

`SettingsStore`（`@Observable`）通过 `UserDefaults` 持久化：
- `serverHost`: 服务器地址（Tailscale IP）
- `serverPort`: 端口（默认 7777）
- `baseURL`: 动态计算 `http://host:port`

## 3. 测试设计

### 3.1 单元测试

| 测试类 | 覆盖内容 |
|--------|---------|
| DashboardAPITests | fetchSessions 成功/失败/URL 非法/服务器错误；sendAction 成功/失败 |
| SessionModelTests | Session / ActionResponse JSON 解码 |
| SettingsStoreTests | 默认值 / baseURL 拼接逻辑 |

Mock 方案：`MockURLProtocol` 拦截 `URLSession` 请求，返回预设响应。

### 3.2 UI 测试

| 测试用例 | 验证点 |
|---------|--------|
| testLaunchShowsNavigationTitle | 启动后显示 "Claude Sessions" |
| testSettingsButtonExists | 设置按钮存在且可访问 |
| testOpenAndCloseSettings | 打开设置 Sheet → 点击完成 → 返回主页 |

Pre-action：scheme 配置自动启动本地 `server.js`（已启动则复用）。

## 4. 项目配置

### xcodegen（project.yml）

```yaml
targets:
  cc-dashboard-ios:          # iOS App
  cc-dashboard-iosTests:     # 单元测试
  cc-dashboard-iosUITests:   # UI 测试

schemes:
  cc-dashboard-ios:
    test:
      preActions:
        - script: |
            if ! curl -sf http://localhost:7777/api/sessions; then
              node "$(SRCROOT)/../../server.js" &
              sleep 3
            fi
```
