# cc-dashboard 功能规格说明（FSD）

## 1. 项目概述

cc-dashboard 是一套用于监控和管理 Claude Code CLI 会话的系统，包含：
- **Web Dashboard**：本地实时监控面板（HTML + Tailwind CSS + Node.js）
- **iOS App**：通过 Tailscale VPN 远程监控和操作

## 2. 功能模块

### 2.1 Web Dashboard

| 模块 | 功能描述 |
|------|---------|
| Session 监控 | 自动读取 `~/.claude/sessions`，展示所有活跃会话 |
| 状态显示 | running（运行中）/ waiting（等待确认）/ idle（空闲） |
| Context 占用 | 可视化进度条展示上下文窗口使用情况 |
| Token 统计 | 累计 Input / Output / Cache Read / Cache Create |
| 最近消息 | 展示最近 10 条交互消息，等待确认消息高亮 |
| 远程确认 | 通过 AppleScript 向 Terminal.app / iTerm2 发送确认/拒绝按键 |
| 实时对话 | SSE 流式接收指定 session 的 transcript 新增消息 |
| 消息发送 | 向终端 session 发送文本消息 |

### 2.2 iOS App

| 模块 | 功能描述 |
|------|---------|
| Session 列表 | 远程获取并展示所有活跃 session |
| 状态排序 | waiting > running > idle，同状态按开始时间倒序 |
| 远程确认 | 点击 waiting session 弹出确认 Sheet，发送 confirm/reject |
| 轮询刷新 | 有 waiting session 时 2 秒轮询，否则 5 秒 |
| 服务器配置 | 设置 Tailscale IP 和端口，测试连接 |
| 下拉刷新 | 支持下拉手动刷新 session 列表 |

## 3. 系统架构

```
┌─────────────┐      HTTP      ┌──────────────┐
│  iOS App    │ ◄────────────► │  Node.js     │
│  (SwiftUI)  │   Tailscale    │  Server      │
└─────────────┘                └──────┬───────┘
                                      │
                              ┌───────┴───────┐
                              │  ~/.claude/   │
                              │  sessions/    │
                              │  projects/    │
                              └───────────────┘
```

## 4. API 接口

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/sessions` | 获取所有活跃 session |
| GET | `/api/sessions/:id/stream` | SSE 实时消息流 |
| POST | `/api/sessions/:id/action` | 发送操作指令（confirm/reject/sendMessage） |

## 5. 数据模型

### Session 状态
- `running`：Claude 正在处理用户消息
- `waiting`：Claude 等待工具执行确认
- `idle`：Claude 已完成回复，等待用户输入

### Session 来源
- `terminal`：Terminal.app / iTerm2
- `vscode`：VS Code 集成终端

## 6. 非功能需求

| 需求 | 指标 |
|------|------|
| 响应时间 | API 响应 < 500ms（本地网络） |
| 兼容性 | iOS 17.0+，macOS Safari / Chrome |
| 并发 | 单用户本地使用，无需考虑高并发 |
| 安全性 | 仅通过 Tailscale 内网访问，无公网暴露 |
