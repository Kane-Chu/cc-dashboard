# cc-dashboard 测试案例

## 1. 单元测试（cc-dashboard-iosTests）

### 1.1 DashboardAPITests

| 用例 | 输入 | 预期结果 |
|------|------|---------|
| fetchSessions 成功 | 返回 200 + 合法 JSON | `[Session].count == 1`, status == .running |
| fetchSessions URL 非法 | baseURL = "not a url" | 抛出 `APIError.invalidURL` |
| fetchSessions 服务器错误 | 返回 500 | 抛出 `APIError.invalidResponse` |
| sendAction 成功 | 返回 `{"success": true}` | `response.success == true` |
| sendAction 业务失败 | 返回 `{"success": false, "error": "..."}` | 抛出 `APIError.serverError` |

### 1.2 SessionModelTests

| 用例 | 输入 | 预期结果 |
|------|------|---------|
| Session 解码 | 完整 JSON 对象 | 所有字段正确映射 |
| ActionResponse 解码 | `{"success": true, "method": "..."}` | success / method / error 正确解析 |
| SessionListResponse 解码 | `{"timestamp": "...", "sessions": [...]}` | timestamp + sessions 数组正确解析 |

### 1.3 AppIconTests

| 用例 | 输入 | 预期结果 |
|------|------|---------|
| 图标不是占位符 | 编译后的 AppIcon PNG | 彩色像素比例 > 5%，非灰白占位符 |

### 1.4 SettingsStoreTests

| 用例 | 输入 | 预期结果 |
|------|------|---------|
| 默认值 | 新实例 | host 为空，port 为 "7777" |
| baseURL 拼接 | host = "100.1.2.3", port = "7777" | `"http://100.1.2.3:7777"` |
| baseURL 空 host | host = "" | 回退到 `"http://localhost:7777"` |
| baseURL 空 port | port = "" | 回退到 `"http://host:7777"` |

## 2. UI 测试（cc-dashboard-iosUITests）

| 用例 | 步骤 | 预期结果 |
|------|------|---------|
| 启动显示标题 | 启动 App | 导航栏显示 "Claude Sessions" |
| 设置按钮存在 | 启动 App | 右上角设置按钮（gear）存在 |
| 打开并关闭设置 | 1. 点击设置按钮<br>2. 点击「完成」 | Sheet 打开 → Sheet 关闭 → 返回主页 |

### 测试环境

- 设备：iPhone 16 Simulator
- 系统：iOS 18.3.1
- 前置条件：本地 server.js 已启动（scheme pre-action 自动处理）

## 3. Web Dashboard 手动测试

| 用例 | 步骤 | 预期结果 |
|------|------|---------|
| Session 列表加载 | 启动 server，打开浏览器 | 显示所有活跃 session 卡片 |
| 状态颜色正确 | 观察不同状态 session | running=绿 / waiting=橙 / idle=灰 |
| 远程确认 | 触发工具调用 → 点击确认 | 终端收到 `1` + Enter |
| SSE 消息流 | 打开对话弹窗 | 实时显示新消息 |
| 消息发送 | 在弹窗输入文字 → 发送 | 终端收到输入文本 |

## 4. 集成测试（iOS + Server）

| 用例 | 步骤 | 预期结果 |
|------|------|---------|
| 连接本地 server | iOS App 默认连接 localhost:7777 | 成功获取 session 列表 |
| 连接 Tailscale | 设置 Tailscale IP → 测试连接 | 成功获取 session 列表 |
| 远程 confirm | 点击 waiting session → 确认 | server 返回 success，状态变为 running |
| 远程 reject | 点击 waiting session → 拒绝 | server 返回 success，状态变为 idle |
