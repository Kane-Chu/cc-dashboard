# cc-dashboard 用户手册

## 1. Web Dashboard

### 1.1 启动

```bash
node server.js
```

默认监听 `http://localhost:7777`。

### 1.2 查看 Session 列表

打开浏览器访问 `http://localhost:7777`，页面自动展示所有活跃 Claude Code 会话。

每个 Session 卡片显示：
- **状态指示灯**：绿色（running）/ 橙色（waiting）/ 灰色（idle）
- **终端来源**：Terminal.app（🖥）/ VS Code（⚡）
- **工作目录**：最后一个路径组件
- **运行时长**：从启动时间计算
- **模型版本**：如 claude-sonnet-4-6
- **Context 占用率**：进度条 + 具体数字

### 1.3 远程确认操作

当 session 状态为 **waiting**（橙色）时：
1. 卡片右上角会显示「确认」和「拒绝」按钮
2. 点击「确认」向终端发送 `1` + Enter
3. 点击「拒绝」向终端发送 `3` + Enter

> 仅支持 Terminal.app 和 iTerm2。VS Code 集成终端需手动操作。

### 1.4 实时对话弹窗

点击卡片右上角的气泡图标：
- 打开居中对话窗口
- 通过 SSE 实时接收该 session 的新消息
- 支持 Markdown 渲染、代码块高亮、图片预览
- 可输入消息并发送（通过 AppleScript 注入终端）

## 2. iOS App

### 2.1 环境准备

1. **电脑端**：
   - 安装并启动 Tailscale
   - 启动 cc-dashboard server：`node server.js`

2. **手机端**：
   - 安装 cc-dashboard-ios App
   - 确保手机和电脑在同一 Tailscale 网络

### 2.2 首次配置

1. 打开 App，点击右上角 **⚙️ 设置**
2. 输入电脑的 **Tailscale IP**（如 `100.x.x.x`）
3. 端口默认 `7777`，无需修改（除非手动指定）
4. 点击 **测试连接** 验证
5. 点击 **完成** 保存

### 2.3 Session 列表

- 按状态排序：**等待确认** > **运行中** > **空闲**
- 同状态下按开始时间倒序
- 下拉可手动刷新
- 列表为空时显示提示：确保电脑上的 Claude Code 正在运行

### 2.4 远程确认

点击状态为 **等待确认** 的 session 卡片：
- 底部弹出 Confirm Sheet
- 显示待确认的 Tool 名称和参数
- 点击「确认」或「拒绝」发送指令
- 操作成功后自动关闭 Sheet 并刷新列表

### 2.5 注意事项

- App 自动轮询：有 waiting session 时每 2 秒刷新，否则每 5 秒
- 首次连接失败时会在页面顶部弹出错误提示
- 确保 Tailscale 在电脑和手机端均已连接
